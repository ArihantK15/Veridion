I got tired of code-review tools that just guess. So I built one that has to show its work.

Aletheore starts as a deterministic repository scanner - no LLM in the scan path, every claim traced to evidence in your repo - and grows into a hosted GitHub App that detects dead code, monitors your API endpoints live, keeps AIRview - an AI-generated map of your architecture - current on every push, and reviews every pull request automatically.

Start Your First Audit Read the Protocol

Why deterministic-first.

No LLM ever runs in the scan path. Aletheore reads your repository with tree-sitter and git log, writes what it found to air.json - the AIR, Aletheore's Intermediate Representation - and every other command - query, diff, the dashboard, the MCP server, even audit's written report - reads from that same file. Nothing downstream states a claim it can't point back to a specific field in it.

How it actually works.

aletheore scan runs once and writes two files: air.json (the canonical record - languages, dependency graph, clusters, git activity, secrets, licenses, vulnerabilities, API endpoints) and air.toon, a TOON-encoded copy of the same data specifically for the MCP server's tool results and the coding-agent adapter audit uses - because that's where token cost is actually paid, not in the file Aletheore itself keeps on disk. Everything downstream - query, diff, the dashboard, the MCP server, the GitHub App - reads the same AIR. One evidence format, one source of truth, one name for it.

What it actually checks.

Every capability below runs from the same air.json - nothing here is aspirational copy.

$ aletheore scan

Deterministic scan

Secrets, dependency vulnerabilities and licenses across 11 languages (Python, JavaScript, TypeScript, Go, Rust, Java, Ruby, PHP, C, C++, C#), API endpoint mapping, architecture clustering, and full git history - all from tree-sitter and git log, no LLM involved.

$ aletheore query dead-code

Dead code detection

Flags unreachable modules and functions from real call-graph analysis - not a guess, a traced absence of any incoming edge from a known entry point.

$ aletheore query hotspots

Git hotspot intelligence

Surfaces the files that change most and who actually owns them, straight from git log - the parts of your codebase most likely to need a careful review.

$ aletheore query database

Database & infrastructure detection

Finds ORM frameworks, migration directories, schema files, Docker Compose services, Kubernetes manifests, and declared environment variable names (never values) - all from static analysis, nothing executed.

on: pull_request

GitHub Action

Posts a PR comment and inline annotations on new secrets, plus a step summary on every run - so review happens where the code already lives.

$ aletheore mcp

MCP server

Gives any coding agent direct, tool-based access to your evidence: imports, dependents, full-text and semantic search, health checks.

$ aletheore index

Semantic code search

A local vector index over your code, embedded with Ollama and queried with cited answers - nothing leaves your machine.

$ aletheore audit

Multi-provider reports

Claude, OpenAI, Gemini, Mistral, Grok, or a fully local Ollama model - bring your own key, with consent shown every single time.

$ aletheore dashboard

Local dashboard

Visualize the dependency graph, clusters, and every finding in a browser tab - reads the same evidence file, nothing else to configure.

Also available as a hosted GitHub App.
